۵SCREEN INVENTORY

فهرست صفحات

۲۲ صفحه/حالت برای MVP و فاز ۲. هر صفحه با محتوای بالای تا (above the fold) و CTA اصلی تعریف شده است.

S01 · خانه (سایت)

هیرو تمام‌عرض با عکس اسکی + نوار KPI (سرعت، ضربان، توان، تعداد آنالیز) + کارت‌های قابلیت + ویدیو نمونه + CTA.

MVP

S02 · قابلیت‌ها

سه ستون: اسکی / دوچرخه / آنالیز ویدیویی؛ هرکدام با نمونهٔ خروجی واقعی.

MVP

S03 · قیمت‌ها و باشگاه

مقایسهٔ ۳ پلن + پرسش‌های متداول + نشان اعتماد. پلن سالیانه هایلایت.

MVP

S04 · ویدیوهای آموزشی

گرید کارت ویدیو با فیلتر رشته؛ embed آپارات/یوتیوب با تصویر جلد.

MVP

S05 · ورود / ثبت‌نام

فرم تک‌ستونه، ورود با کد یکبارمصرف؛ حداکثر ۲ فیلد در گام اول.

MVP

S06 · داشبورد اپ

سلام شخصی + ۴ کارت KPI + نمودار روند + بینش‌های AI + آخرین آنالیزها.

MVP

S07 · لیست آنالیزها

ردیف‌ها با وضعیت (در صف / آماده) + حالت خالی با CTA «اولین آنالیز».

MVP

S08 · ویزارد آنالیز جدید

۴ گام با نوار پیشرفت: رشته → راهنمای زاویه → آپلود → انتخاب پلن.

MVP

S09 · راهنمای زاویهٔ فیلم‌برداری

تصاویر درست/غلط + چک‌لیست ۴ موردی؛ بخشی از ویزارد و قابل دسترسی مستقل.

MVP

S10 · نتیجهٔ آنالیز

پلیر ویدیو با نشانه‌گذاری زمانی + کارت نکات مربی + امتیاز تکنیک + CTA اشتراک.

MVP

S11 · رزرو کلاس — لیست مربی

کارت مربی: عکس، تخصص، امتیاز، قیمت جلسه.

MVP

S12 · رزرو کلاس — تقویم

تقویم شمسی، سانس‌های خالی سبز؛ بنر «جلسهٔ اول VIP رایگان».

MVP

S13 · پرداخت

خلاصهٔ سفارش + درگاه؛ حالت‌های موفق/ناموفق با مسیر بازیابی.

MVP

S14 · پروفایل و اشتراک

وضعیت پلن، تاریخ تمدید، سوابق پرداخت، تنظیم اعلان‌های ایمیلی.

MVP

S15 · تمرین دوچرخه

برنامهٔ هفتگی + حلقهٔ پیشرفت + رکوردهای شخصی.

فاز ۲

S16–S22 · تکمیلی

اعلان‌ها، جزئیات ویدیو، ویرایش پروفایل، ۴۰۴/۵۰۰، حالت آفلاین، تأیید ایمیل، صفحهٔ VIP.

۹.۵ — دسترس‌پذیری (WCAG AA) و RTL

چک‌لیست دسترس‌پذیری

✓ ناحیهٔ لمس ≥ ۴۴×۴۴px با فاصلهٔ ≥ ۸px ✓ فوکوس نمایان (رینگ ۲px آبی) ✓ ترتیب کیبورد = ترتیب بصری ✓ لیبل صفحه‌خوان برای دکمه‌های آیکنی ✓ خطای فرم زیر فیلد + aria-live ✓ جدول جایگزین برای نمودارها ✓ سلسله‌مراتب هدینگ بدون پرش

قواعد RTL

✓ استفاده از ویژگی‌های منطقی CSS (margin-inline-start به‌جای left/right) ✓ آینه‌شدن آیکن‌های جهت‌دار (فلش‌ها) و نه آیکن‌های خنثی ✓ اعداد فارسی در متن، لاتین در نمودار و کد ✓ اسلایدر و پیشرفت از راست به چپ ✓ تست هر کامپوننت در هر دو جهت

۱۰FRONTEND ARCHITECTURE

معماری فرانت‌اند

Next.js با App Router؛ صفحات بازاریابی سرور-رندر برای سئو، اپ ورزشکار کلاینت-محور با کش داده. RTL و فارسی از روز اول در معماری، نه به‌عنوان وصله.

۱۱DESIGN-TO-CODE PLAN

برنامهٔ طراحی تا کد

یک منبع حقیقت: توکن‌های JSON → متغیرهای CSS → Tailwind → کامپوننت‌ها. هیچ مقدار خامی (hex یا px دلخواه) داخل کامپوننت‌ها مجاز نیست.
